# 大模型流式输出：问题与解决（含代码路径）

## 问题

前端调用对话接口后，为什么有时看不到“逐字输出”，或者访问 `http://localhost:8000/` 只看到：

```json
{"detail":"Not Found"}
```

## 结论（先说答案）

1. 根路径 `/` 返回 `Not Found` 是正常现象，流式接口不在 `/`。  
2. 本项目的流式输出走的是 `POST /api/chat`，后端用 LiteLLM 的 `stream=True` 拉取分块，再逐块推给前端。  
3. 当前实现是“提取 chunk 中的文本再输出”，不是把完整 OpenAI chunk JSON 原样透传。

## 代码在哪里

### 1) 前端请求与读取流

- 请求发起：`frontend/src/lib/chatApi.ts:14`
- 读取流：`frontend/src/lib/chatApi.ts:38`
- 每次 chunk 回调更新：`frontend/src/lib/chatApi.ts:49`

关键逻辑（前端）：

```ts
const response = await fetch(`${API_BASE_URL}/api/chat`, { method: 'POST', ... })
const reader = response.body.getReader()
while (true) {
  const { done, value } = await reader.read()
  if (done) break
  const text = decoder.decode(value, { stream: true })
  if (text) options.onChunk(text)
}
```

### 2) 前端转发到大模型服务

- 路由重写：`frontend/next.config.js:11`

`/api/chat` 会被代理到 `${LLM_API_PROXY_TARGET}/api/chat`（默认 `http://localhost:8000/api/chat`）。

### 3) 大模型后端流式生成

- 接口入口：`llm_api/app/main.py:95`
- 开启 LiteLLM 流式：`llm_api/app/main.py:74`
- 发起流式请求：`llm_api/app/main.py:82`
- 逐块产出：`llm_api/app/main.py:84`
- 返回 StreamingResponse：`llm_api/app/main.py:110`

关键逻辑（后端）：

```py
completion_kwargs = {
    "model": model_name,
    "messages": model_messages,
    "stream": True,
    "temperature": payload.temperature
}
stream = await acompletion(**completion_kwargs)
async for chunk in stream:
    text = _extract_text_from_chunk(chunk)
    if text:
        yield text
```

### 4) OpenAI chunk 文本提取

- 提取函数：`llm_api/app/main.py:28`

该函数会从 chunk 中优先取：
- `choices[0].delta.content`
- 或 `choices[0].message.content`

所以你看到的是纯文本增量，而不是完整 JSON chunk。

## 解决方式（按问题排查）

### 场景 A：访问 8000 显示 Not Found

这是因为你访问了错误路径 `/`。请用：

```bash
curl http://127.0.0.1:8000/health
curl -N -X POST http://127.0.0.1:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"你好"}]}'
```

### 场景 B：前端没有逐字输出

1. 检查前端是否请求 `/api/chat`：`frontend/src/lib/chatApi.ts:14`  
2. 检查代理是否指向 8000：`frontend/next.config.js:11`  
3. 检查后端是否启用 `stream=True`：`llm_api/app/main.py:74`  
4. 检查后端是否正确 `yield text`：`llm_api/app/main.py:87`

### 场景 C：想要原样 OpenAI chunk JSON

当前代码做了“文本抽取”。  
如果要原样返回 JSON chunk，需要改 `llm_api/app/main.py` 中 `yield text` 的方式，改成输出 chunk 的序列化结果（例如 SSE 或 JSONL）。

## 一句话理解

本项目的“流式输出”是：**模型边生成，后端边转发文本片段，前端边读边显示**。
